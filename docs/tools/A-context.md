# [A] Plugin Context — Управление состоянием

## Назначение

Сервер запоминает текущий плагин между вызовами инструментов.
Это позволяет ADB-инструментам знать file_path и plugin_id без повторного ввода,
а генераторам хуков — знать, что уже зарегистрировано.

---

## Инструменты

### set_plugin_context

Устанавливает текущий рабочий плагин.

**Параметры:**
```
plugin_id     string  — уникальный ID плагина (2-32 символа)
plugin_name   string  — отображаемое имя
file_path     string  — локальный путь к .py файлу
device_serial string? — серийный номер ADB-устройства (если несколько)
```

**Валидация plugin_id:**
- Длина 2–32 символа
- Начинается с буквы (a-z)
- Содержит только: latin letters, digits, `_`, `-`
- Regex: `/^[a-z][a-z0-9_-]{1,31}$/`

**Возвращает:** подтверждение + результат валидации. Если ID невалиден — список ошибок
без сохранения.

---

### get_plugin_context

Возвращает текущий контекст целиком.

**Параметры:** нет

**Возвращает:**
```
plugin_id, plugin_name, file_path, device_serial,
registered_hooks (список), metadata (все поля)
```

---

### clear_plugin_context

Очищает текущий контекст.

**Параметры:** нет

**Возвращает:** подтверждение.

---

### validate_plugin_metadata

Проверяет корректность метаданных плагина без сохранения.

**Параметры:**
```
id           string
name         string
description  string?
author       string?
version      string?
icon         string?
app_version  string?   — должен быть формата ">=X.Y.Z"
sdk_version  string?   — должен быть формата ">=X.Y.Z"
requirements string[]? — PEP 508 строки
```

**Проверки:**
- `__id__`: regex `/^[a-z][a-z0-9_-]{1,31}$/`
- `__version__`: semver формат если указан
- `__app_version__`: оператор сравнения + версия (пример: `>=12.5.1`)
- `__sdk_version__`: оператор сравнения + версия (пример: `>=1.4.3.6`)
- `__requirements__`: каждая строка — валидный PEP 508 спецификатор

**Возвращает:** `"OK"` или список ошибок с описанием.
